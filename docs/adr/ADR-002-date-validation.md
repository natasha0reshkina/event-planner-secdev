# ADR-002: Валидация даты события
Дата: 2025-10-20
Статус: Accepted

## Context
В Event Planner нельзя создавать события в прошлом. При сравнении с текущим моментом появляются ошибки, когда событие на сегодняшний день считается прошлым. Требуется нормализовать сравнение.

## Decision
Дата события сравнивается с началом текущего дня по часовому поясу сервера. Если дата меньше начала дня, возвращается код 422. Сегодня и будущее считаются допустимыми.

Проверка:
отклонение прошлых дат не ниже девяноста пяти процентов на наборе тестов,
модульные и сквозные тесты проверяют даты в прошлом, настоящем и будущем.

## Consequences
Предсказуемое поведение для событий на сегодня. Нужно зафиксировать часовой пояс и добавить тесты на переходы суток.
